# Royal Vault — Desktop App

A native Mac desktop app for The Royal Kat & Kai Vault.
Built with Electron. Connects to the existing backend at `https://royalkvault.up.railway.app`.

---

## Prerequisites

- **Node.js 18+** (`node -v` to check)
- **npm 9+** (bundled with Node)
- macOS (Apple Silicon or Intel — both supported)

---

## Development

```bash
# 1. Enter the vault-desktop directory
cd vault-desktop

# 2. Install dependencies (also runs postinstall to copy socket.io vendor file)
npm install

# 3. Generate placeholder icons
npm run create-icons

# 4. Launch in development mode
npm start
```

The app opens a frameless window with the Royal Vault UI.
Hot-reload is NOT enabled — restart the app after code changes (`Ctrl+C` then `npm start`).

---

## First-Run Auth Flow

1. **Site password** — enter the vault access password once (stored securely for future launches).
2. **Profile select** — choose Kaliph or Kathrine.
3. **4-digit PIN** — enter your profile PIN.

On subsequent launches the app skips to step 3 (for the last-used profile).

## Building for Distribution

```bash
# Generate icons first
npm run create-icons

# Build .dmg + .zip (universal binary — works on M1/M2/Intel)
npm run build:mac
```

Output appears in `dist/`:
- `Royal Vault-1.0.0.dmg` — drag-to-Applications installer
- `Royal Vault-1.0.0-mac.zip` — bare .app archive

> **Note:** Code signing is disabled. macOS Gatekeeper will warn on first open — right-click the .app → Open to bypass.

## Replacing Icons

The placeholder icons are generated by `scripts/create-icons.js`.
To use real artwork:

1. Create a **1024×1024 PNG** and save it to `build/icon.png`
2. Create a **32×32 black-on-transparent PNG** for the menu bar and save to `build/tray.png`
3. `electron-builder` will auto-convert `icon.png` → `icon.icns` during build

For best Retina quality, also add `build/icon@2x.png` (2048×2048).

## Architecture Notes

- **Auth:** The site password is stored (encrypted) in `electron-store` after first entry. Profile PIN is verified via `/api/auth/profile`. The Electron persistent session (`persist:royalvault`) stores cookies between launches.
- **CORS:** The main process injects `Access-Control-Allow-Origin: *` headers on responses from the Railway backend, so `fetch()` calls work from the renderer without a proxy.
- **Socket.IO:** Loaded from `renderer/vendor/socket.io.min.js` (copied from `node_modules` during `npm install`). Connects with `withCredentials: true`.
- **Notifications:** Renderer emits IPC events → main process shows native `Notification`.
- **Window close:** Hides to tray (app keeps running). Cmd+Q or tray → Quit to actually exit.
- **Security:** `contextIsolation: true`, `nodeIntegration: false`. All Node.js access is via `contextBridge` in `preload.js`.

## Keyboard Shortcuts

| Shortcut | Action |
|---|---|
| `Cmd+Enter` | Send message |
| `Escape` | Cancel reply / edit / close pickers |
| `↑` (in empty input) | Edit your last message |
| `Cmd+1–7` | Switch sections (Chat, Notes, Calendar…) |
| `Cmd+Q` | Quit app |
